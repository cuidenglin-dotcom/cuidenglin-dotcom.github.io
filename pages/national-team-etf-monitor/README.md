# 国家队 ETF 份额监控：维护说明

本目录是一个可独立部署的静态页面。页面会在浏览器中读取同目录的 `etf_history.csv`，无需数据库或服务端程序。

## 日常更新（推荐）

每个工作日只替换 `etf_history.csv`，不要修改 `index.html`。CSV 应保留完整历史记录，而不是只放最新一天的四条记录。页面会自动识别最新日期、绘制趋势、切换全部/单只 ETF 视图并分页。

标准表头必须完全一致：

```csv
date,code,name,share_change_100m,share_balance_100m,prev_date,prev_share_balance_100m,status,source,updated_at
```

要求：

- 文件编码：UTF-8（建议无 BOM）。
- `date`：`YYYYMMDD`，例如 `20260717`。
- 数值字段：使用英文小数点，不带“亿份”等单位文字。
- 每个日期通常有四个代码：`510300.SH`、`510330.SH`、`510310.SH`、`159919.SZ`。
- 空值可保留为空，但字段分隔逗号不能省略。
- `status` 推荐为 `ok`；异常状态会在页面上提示。
- 同一日期与代码不要出现重复记录。

更新后至少检查：CSV URL 返回 200；网页最新日期正确；四张最新卡片、图表和历史表均能显示；手机宽度下无横向溢出。

## 数据读取顺序

页面优先读取 `./etf_history.csv`。读取失败或 CSV 校验失败时，会退回 HTML 内嵌的 2026-07-13 至 2026-07-17 示例数据，并明确标注“备用数据”。因此，日常更新成功与否应以页面的数据来源提示和最新日期为准。

## 页面代码维护

- 数据文件路径由 `index.html` 内的 `DATA_URL` 控制。
- ETF 显示顺序由 `ETF_ORDER` 控制。
- 每页条数默认值在 `state.pageSize` 中设置，界面也允许切换。
- 三个前端库从站内 `../../assets/vendor/` 加载；版本和校验值见 `../../assets/vendor/DEPENDENCIES.md`。
- 页面没有 CSV 上传按钮，也没有滚动动画；数据应由网站后台、部署流程或后续同步脚本写入。

## 回滚

更新前保留旧 CSV。若新数据导致页面异常，将旧文件恢复为 `etf_history.csv` 即可；无需改动 HTML 或重启 Web 服务。
